## PATTERN EXTRACTION METHODOLOGY

### Source Material

**Document:** PAPER2_V2_MASTER_SOURCE_BUILD.md
- **Length:** 1,325 lines, ~8,500 words
- **Content:** Energy-regulated population homeostasis and timescale-dependent constraint manifestation in NRM
- **Sections:** Abstract, Introduction, Methods, Results, Discussion, Conclusions, References
- **Key Experiments:** C168-170 (bistability), C171 (baseline homeostasis), C176 V6 (multi-scale validation)

### Extraction Protocol

**Step 1: Systematic Reading**
- Read manuscript section-by-section (Abstract → Conclusions)
- Identify explicit findings, methodologies, principles, and meta-insights
- Note line numbers for precise source location

**Step 2: Pattern Categorization**
- **SF (Scientific Findings):** Quantitative results, measurements, thresholds, empirical observations
- **MP (Methodological Principles):** Experimental protocols, validation strategies, statistical approaches
- **FP (Framework Principles):** NRM, Self-Giving, Temporal Stewardship theoretical principles
- **MR (Meta-Research Insights):** Lessons learned, methodological recommendations, research process reflections

**Step 3: 8-Dimension Coding**

For each pattern, code:

1. **Pattern_ID:** Unique identifier (e.g., P2-SF-001)
2. **Content:** Direct excerpt or paraphrased finding from paper
3. **Category:** SF/MP/FP/MR
4. **Format:**
   - C = Code (implementation detail)
   - D = Documentation (framework docs)
   - P = Paper (manuscript text)
   - M = Multiple sources
5. **Precision:**
   - Q = Quantitative (numerical, measurable)
   - L = Qualitative (descriptive, conceptual)
   - X = Mixed (both quantitative and qualitative)
6. **Transparency:**
   - E = Explicit (directly stated)
   - I = Implicit (inferred from context)
   - M = Meta (reflection on encoding itself)
7. **Framework:**
   - N = NRM (Nested Resonance Memory)
   - S = Self-Giving Systems
   - T = Temporal Stewardship
   - R = Reality Imperative
   - U = Universal (applies across frameworks)
8. **Function:**
   - TD = Training Data (encodes findings for future AI)
   - ML = Methodological Lesson (teaches research methods)
   - FV = Framework Validation (supports theoretical framework)
   - PT = Pattern Template (provides reusable template)

**Step 4: Source Location Documentation**
- Record first/last occurrence (cycle or experiment)
- Note source file and line number for traceability

## THREATS TO VALIDITY (PATTERN EXTRACTION)

### Threat 1: Extraction Bias

- **Risk:** Pattern selection influenced by researcher's expectations
- **Mitigation:** Systematic section-by-section extraction, explicit coding criteria
- **Residual Risk:** Moderate (subjective judgment in categorization)

### Threat 2: Pattern Granularity

- **Risk:** Some patterns too coarse (lumping) or too fine (splitting)
- **Example:** "Multi-scale validation" as single pattern vs 3 separate patterns for each timescale
- **Mitigation:** Use consistent granularity (one pattern = one distinct finding/principle)
- **Residual Risk:** Low (consistent heuristic applied)

### Threat 3: Incomplete Coverage

- **Risk:** Some patterns in Paper 2 not extracted
- **Mitigation:** Systematic reading of all sections (Abstract → Conclusions)
- **Validation:** Phase 6 Discoverability Experiment will test if AI finds patterns missed in manual extraction
- **Residual Risk:** Moderate (manual extraction may miss implicit patterns)

### Threat 4: Coding Consistency

- **Risk:** 8-dimension coding applied inconsistently across patterns
- **Example:** SF vs MP boundary (methodological finding vs principle?)
- **Mitigation:** Explicit coding definitions, representative examples documented
- **Residual Risk:** Low (definitions clear, examples provided)
