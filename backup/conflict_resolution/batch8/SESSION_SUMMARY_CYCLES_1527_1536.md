## MAJOR DISCOVERIES

### 1. Multi-Trophic Coexistence Zone (C295)

**Goldilocks zone for predator-prey:**
- 0.001: Predators starve (0%)
- 0.002-0.003: Coexistence possible (67-100%)
- 0.01: Overexploitation collapse (0%)

### 2. Counterintuitive Spatial Effects (C296-C297)

**Refuge can destabilize:**
- High attack rate: Refuge always harmful
- Intermediate attack rate: U-shaped effect (0-10%: good, 10-40%: bad, >40%: partial recovery)

"Protection" can worsen outcomes.

### 3. Type II Stabilization (C298)

**Predator satiation rescues collapse:**
- Type I: 0% coexistence
- Type II (h=0.02): 67% coexistence

Handling time prevents overexploitation.

### 4. Antagonistic Mechanism Interaction (C299)

**Combined mechanisms can be worse:**
- Type II alone: 33%
- Type II + refuge: 0%

Stabilizing mechanisms can cancel each other.

### 5. Evolution-Ecology Hierarchy (C300-C303)

**Ecology constrains evolution:**
- Attack rate 0.01 (unstable): Evolution doesn't help
- Attack rate 0.003 (optimal): Evolution doubles coexistence (33% → 67%)

"Evolution cannot save what ecology has condemned."

## THEORETICAL PRINCIPLES

### 1. Parameter Space Topology

- Narrow viable zones
- Valleys of death
- Non-linear boundaries
- Context-dependent effects

### 2. Mechanism Interactions

Combining mechanisms can be:
- Synergistic (better than sum)
- Additive (equal to sum)
- Antagonistic (worse than sum)

Must be tested, not assumed.

### 3. Ecology-Evolution Hierarchy

1. **Primary:** Ecological parameters (attack rate, K, Type II)
2. **Secondary:** Evolutionary dynamics (mutation, selection)

Set ecology first, then evolution optimizes.

## DESIGN PRINCIPLES

### For Predator-Prey Coexistence

**Recommended:**
1. Attack rate in Goldilocks zone (0.002-0.003)
2. Type II functional response
3. If using refuge: None OR substantial (>40%)
4. Maintain genetic diversity

**Avoid:**
1. Intermediate interventions (10-30% refuge)
2. Combining multiple reduction mechanisms
3. Relying on evolution to fix parameter problems

### For System Design

1. Establish viable parameter space first
2. Add evolution as optimization layer
3. Test intervention combinations
4. Expect non-linear responses

## KEY INSIGHTS

### 1. Ecology is Foundation, Evolution is Optimization

Evolutionary processes cannot create viable systems from scratch. They can only optimize within ecologically viable parameter space.

### 2. Interventions Require Careful Testing

Simple intuitions fail:
- "More refuge = more stable" → FALSE
- "Combining mechanisms = better" → FALSE
- "Evolution will rescue the system" → FALSE (unless viable)

### 3. Parameter Choice is Critical

The single most important decision is choosing parameters in viable range. Everything else is secondary.
